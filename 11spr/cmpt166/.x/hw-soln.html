HW1 (due Tue 18 Jan) (10pts)

  1. Compare and contrast the JDK and the JRE.
    Java Development Kit: JRE plus compiler (javac) for writing your own Java programs.
    Java Runtime Environment: only the Java VM (java) for running precompiled Java programs.
  2. Compare and contrast compiled vs. interpreted languages.
    Compiled: source code is compiled, linked, and assembled before being executed (run). Examples include C/C++, Java, M2, etc.
    Interpreted: source code is compiled on-the-fly, allows for interactive sessions like in IDLE. Examples include Python, Perl, JavaScript, etc.
  3. What do comments look like in Java?
    Either C-style: /* comment */, or C++-style: // comment
  4. Write a minimal "Hello, World!" program in Java. 
      	public class HelloWorld {
		public static void main( String args[] ) {
			System.out.println("Hello, World!");
		}
	}
  5. Describe the eight primitive types in Java, and say something about their differences.
    boolean: either true or false (1byte).
    char: a single Unicode character (2bytes).
    byte, short, int, long: integers of various sizes (1, 2, 4, 8 bytes).
    float, double: floats of various precisions (4, 8 bytes).
  6. What is the difference between an expression and a statement? Give an example of each.
    Expression: returns a result for use in a larger expression or statement. Example: 2 + 3
    Statement: stands alone as a command to be executed. Example: x = 2 + 3

  5. Explain the model-view-controller pattern when applied to a typical GUI. [3]
    The model-view-controller pattern when applied to the task of designing a GUI allows the task to be divided into two main subtasks: 1) Designing and coding the appearance of the GUI on the screen; 2) Designing and coding the actions performed in response to button clicks and other user actions. This approach allows GUI components to send messages to determine which actions should be taken.
  6. Name and describe all the classes involved with writing a serializable object out to the file "output.txt". [3]
    File takes the string filename and looks it up in the filesystem.
    FileOutputStream takes the File object and opens the file for writing binary data.
    ObjectOutputStream takes the FileOutputStream and prepares it for writing out serialized objects.

HW5 (due Thu 7 Apr) (20pts)

  1. What is an Android Activity? Is it the same as an application? Discuss the relationship between an Activity and an app. [5]
    An Activity presents the user interface for a single task; e.g., a single screen or dialog in an application. Apps generally are composed of several Activities, and/or BackgroundServices, ContentProviders, BroadcastListeners.
  2. Name and describe the four states of an Activity's lifecycle. [6]
    Active (running in foreground), paused (e.g., blocked by a pop-up dialog), stopped (no longer visible), dead (removed by OS)
  3. Describe how to design an Android application which is internationalization-ready (i.e., its text can be easily translated). [5]
    Put all user-visible strings in a separate strings.xml file and refer to them in your code (or object properties) only via resource IDs. To create a translation, make another version of the strings.xml which contains the translated strings. Then the system can select the appropriate version of the strings at runtime.
  4. Name and describe two Creational design patterns from the "Gang of Four" (GoF) book. [4]
    Factory Method (abstract constructor), Abstract Factory (hierarchy of factory classes to create objects), Builder (director delegates construction to a set of builders), Prototype (clone an existing object to make more), and Singleton (only one instance of the class is allowed).
